集合包

发表于 2015-08-04   |   分类于 Java   |  

JDK类库：集合包（java.util）

Collection

Collection接口：单对象存储的集合对象

List

允许元素重复

ArrayList

实现方式：数组方式存放对象

  • 创建：ArrayList(int)
    实例化一个大小为intitailCapacity（默认为10）的Object数组，并赋值给elementData
  • 插入对象：add(E)
    • ArrayList已有元素数量+1，赋值给minCapacity
    • 若minCapacity>Object.length，则将Object赋给一个数组对象，产生一个新的数组容量值（大小为当前Object.length*1.5+1）
    • 若新的Object.length仍小于minCapacity，则以minCapacity作为新容量值
    • 调用Arrays.copyOf生成新的数组对象，若想调整容量的增长策略，可继承ArrayList并覆盖ensureCapacity方法
    • Arrays.copyOf实现方法：（未完。。。）
  • 删除对象：remove(E)
  • 获取单个对象：get(int)
  • 遍历对象：iterator()
  • 判断对象是否存在：contains(E)

注意要点：

  • ArrayList基于数组实现，无容量限制
  • 插入元素时可能要扩容，删除时不会减少容量（可调用trimToSize()缩小容量）
  • 查找元素时要遍历数组，对于非null元素采用equals方式查找
  • ArrayList是非线程安全的

LinkedList

实现方式：基于双向链表，Entry类代表集合中的元素

  • 创建：LinkedList()
  • 插入对象：add(E)
    • 创建新的Entry对象，修改相邻两个元素的属性
  • 删除对象：remove(E)
  • 获取单个对象：get(int)
    • 判断传入的index是否在size范围内，抛出IndexOutOfBoundsException
    • 判断index是否小于size的一半，若小于，则从头向后找，否则从尾部向前找
  • 遍历对象：iterator()
  • 判断对象是否存在：contains(E)

注意要点：

  • LinkedList基于双向链表实现
  • 插入元素时要创建新的Entry对象，并修改前后元素的引用
  • 查找删除元素时要遍历链表
  • LinkedList是非线程安全的

Vector

实现方式：Vector和ArrayList一样，基于Object数组实现

  • 创建：Vector()
    • 默认创建大小为10的Object数组，将capacityIncrement设置为0
  • 插入对象：add(E)
    • add方法是线程安全的，有synchronized关键字
    • 扩大数组方式：
      • 若capacityIncrement>0，则将Object数组大小扩大为现有size+capacityIncrement
      • 若capacityIncrement≤0，则将Object数组大小扩大为现有size的两倍
    • 其他和ArrayList基本相同
  • 删除对象：remove(E)
    • 有synchronized关键字，实现和ArrayList完全相同
  • 获取单个对象：get(int)
    • 有synchronized关键字，实现和ArrayList完全相同
  • 遍历对象：iterator()
    • 实现和ArrayList完全相同
  • 判断对象是否存在：contains(E)
    • indexOf方法有synchronized关键字

注意要点：

  • Vector是基于Synchronized实现的线程安全的ArrayList
  • 容量扩充机制和ArrayList不同，可通过传入capacityIncrement控制容量扩充

Stack

实现方式：Stack继承自Vector，在Vector基础上实现了LIFO的弹出及压入操作

  • push：
    • 调用Vector中的addElement
  • pop：
    • 调用peek获取元素，同时删除数组的最后一个元素
  • peek：
    • 获取当前Object数组大小，并获取数组最后一个元素

注意要点：

  • Stack基于Vector实现，支持LIFO（后进先出）

Set

不允许元素重复
基于HashMap实现

HashSet

实现方式：Set接口的实现，基于HashMap实现

  • 创建：HashSet(int)
    • 创建一个HashMap对象
  • 插入对象：add(E)
    • 调用HashMap的put(Object, Object)方法
  • 删除对象：remove(E)
    • 调用HashMap的remove(E)方法
  • 判断对象是否存在：contains(E)
    • 调用HashMap的containsKey(E)方法
  • 遍历对象：iterator()
    • 调用HashMap的keySet的iterator方法
    • 不支持get(int)获取单个对象，只能通过iterator方法获取

注意要点：

  • HashSet基于HashMap实现，无容量限制
  • HashSet是非线程安全的

TreeSet

实现方式：TreeSet基于TreeMap实现，TreeSet和HashSet主要不同在于TreeSet对于排序的支持（如传入Comparator实现、descendingSet及descendingIterator等）

  • 创建：TreeSet(int)
    • 创建一个TreeMap对象
  • 插入对象：add(E)
    • 调用TreeMap的put(Object, Object)方法，要增加的元素作为key，之前已创建的一个final的Object对象作为value
  • 删除对象：remove(E)
    • 调用TreeMap的remove(Object)方法
  • 遍历对象：iterator()
    • 调用TreeMap的navigableKeySet的iterator方法
    • 不支持get(int)获取单个对象，只能通过iterator方法获取

注意要点：

  • TreeSet基于TreeMap实现，支持排序
  • TreeSet是非线程安全的

Map

Map接口：key-value的集合对象

HashMap

实现方式：

  • 创建：HashMap()
    • loadFactor=0.75，threshold=12，创建大小为16的Entry对象数组
    • 若传入初始容量值initialCapacity，则capacity大小为大于initialCapacity的2的倍数
  • 插入对象：put(Object key, Object value)
    • key为null时：
      • 基于Entry的next属性遍历，若存在key，则赋值其value，否则头插法创建新Entry对象
    • key不为null时：
      • 先获取key本身的hashCode，再对hashCode做hash操作
      • hash出来的值与Entry对象数组的大小减1的值进行按位与，得出当前key要存储到数组的位置
      • 获取数组对应的Entry对象，调用Entry的next遍历，寻找hash值和计算出的hash值相等，且key相等或equals的Entry对象
      • 若找到，则替换Entry对象的值，完成put操作，并返回旧的值
      • 否则，则向对应的数组位置利用头插法增加新的Entry对象（hash冲突使用链表法解决）
    • 扩容机制：插入后Entry数组中已用大小≥threshold，则将Entry数组扩大为当前大小的两倍，扩大时对当前Entry对象数组中的元素重新hash，并填充数组，最后重新设置threshold值
  • 获取单个对象：get(Object key)
    • key为null时：
      • 获取数组中第一个Entry对象，基于next遍历，查找key为null的Entry对象，若找到，返回Entry的value，否则返回null
    • key不为null时：
      • 对key进行hash和按位与操作，找到对应的存储位置
      • 获取此位置对应的Entry对象，基于next遍历，查找hash值和计算出的hash值相等，且key相等或equals的Entry对象，若找到，返回其value，否则返回null
  • 删除对象：remove(Object key)
    • 和get类似，找到匹配的key后，若数组上的元素等于key，则将数组元素的值设为其next元素的值；若数组上的元素不等于key，则对链表遍历，直到找到匹配的key或链表尾
  • 判断对象是否存在：contains(Object key)
    • 调用getEntry方法，getEntry过程和get基本相同，只是在找到匹配的key后直接返回Entry对象，而containsKey判断返回的对象是否为null，返回true/false
  • 遍历对象：keySet()
    • 调用keySet后会返回一个HashMap中的KeySet对象实例
    • 调用iterator时返回一个KeyIterator对象实例
    • 调用next时，遍历整个数组及Entry对象的链表
    • HashMap遍历时无法保证顺序

注意要点：

  • HashMap采用数组方式存储key-value构成的Entry对象，无容量限制
  • HashMap基于key hash寻找Entry对象存放到数组的位置，hash冲突采用链表的方式解决
  • 插入元素时可能会扩大数组容量，此时需要重新计算hash并复制对象到新的数组
  • HashMap是非线程安全的，并发场景中可能在执行get时进入死循环，将CPU消耗到100%
  • 并发场景中推荐使用并发包中的ConcurrentHashMap

TreeMap

实现方式：支持排序的Map实现

  • 创建：TreeMap()
    • 将comparator置为null，也可使用带Comparator参数的构造器
  • 插入对象：put(Object key, Object value)
    • 先判断root是否为null
    • root为null：创建新的Entry对象，赋值给root
    • root不为null：
      • 若传入了指定的Comparator实现，则基于红黑树的方式遍历，基于comparator比较key应在树的左边还是右边
        • 若找到相等的key，则直接替换其value，并返回
        • 若未找到相等的key，则一直找到结点为null的元素
      • 若Comparator实现为null，则判断key是否为null，若key为null则抛出NullPointerException
        （未完）
    • TreeMap基于红黑树实现，因此要求一定要有key比较的方法，传入Comparator实现或key对象实现Comparable接口
  • 获取单个对象：get(Object key)
    • TreeMap查找key时是红黑树查找过程，从根开始往下比较，一直找到相等的key，并返回其value
  • 删除对象：remove(Object key)
    • 首先调用getEntry方法，然后将Entry从红黑树删除，并调整树上相关结点
  • 判断对象是否存在：contains(Object key)
    • 调用getEntry方法，判断返回的Entry对象是否为null，返回true/false
  • 遍历对象：keySet()
    • 调用keySet后返回TreeMap的内部类KeySet对象的实例，iterator遍历从根开始，基于红黑树顺序完成

注意要点：

  • TreeMap基于红黑树实现，无容量限制
  • TreeMap是非线程安全的
